# Homework 3
## a) Utility Class
```
System.IO ( System.IO.Path )
```
### สาเหตุที่จัดอยู่ในกลุ่ม Utility Class
เป็น Class ที่ใช้เป็นเครื่องมือช่วยจัดการ path ของไฟล์ และโฟลเดอร์ โดยเป็น static class ที่ไม่ต้องสร้างออบเจกต์ และไม่มีการเก็บสถานะภายใน คลาสนี้ทำหน้าที่เพียงรับค่า Input และประมวลผลเพื่อคืนผลลัพธ์

- ไม่ใช่ Mutable เพราะ ไม่มี object ให้แก้ค่า
- ไม่ใช่ Immutable เพราะ ไม่ได้แทนข้อมูลที่ต้องคงสภาพ

### ลักษณะการใช้งาน  
Path มีคุณสมบัติสำคัญ คือ เป็น static class ไม่สามารถสร้าง object ( new Path() ทำไม่ได้ ) มีแต่ Method ช่วยเหลือ เช่น
```
Combine

GetExtension

GetFileName
```

ตัวอย่างโค้ด:
```
string fullPath = Path.Combine("C:\\Data", "file.txt");
string ext = Path.GetExtension(fullPath);
```
- เรียกใช้ได้ทันที ไม่ต้องมี instance
- ไม่มีข้อมูลภายในที่เปลี่ยนค่า

## b) Mutable Class
```
System.Collections.Concurrent
```
### สาเหตุที่จัดอยู่ในกลุ่ม Mutable Class
เป็น namespace ที่ประกอบด้วยคลาสประเภท collection ซึ่งสามารถเปลี่ยนแปลงข้อมูลภายในได้ตลอดเวลา หลังจากสร้าง Object แล้ว เช่น การเพิ่ม หรือลบข้อมูล โดยมีการออกแบบให้รองรับการใช้งานพร้อมกันหลาย thread-safe

- ไม่ใช่ Utility เพราะ คลาสใน System.Collections.Concurrent ต้องสร้าง object เก็บข้อมูลภายใน และมี lifecycle
- ไม่ใช่ Immutable เพราะ สามารถเพิ่ม, ลบ, แก้ไขข้อมูล ใน object เดิมได้ และทำให้ปลอดภัยต่อหลาย thread

### ลักษณะการใช้งาน  
ตัวอย่างคลาส:
```
ConcurrentDictionary<TKey,TValue>

ConcurrentQueue<T>

ConcurrentBag<T>
```
ตัวอย่างโค้ด:
```
var dict = new ConcurrentDictionary<int, string>();
dict.TryAdd(1, "One");
dict.TryAdd(2, "Two");
```
- ข้อมูลภายใน dict เปลี่ยนค่าได้หลังจากสร้าง object แล้ว
- หมายความว่า object มี state ที่เปลี่ยนแปลงได้

## c) Immutable Class
```
System.Collections.Frozen
```
### สาเหตุที่จัดอยู่ในกลุ่ม Immutable Class
เป็น namespace ที่ประกอบด้วยคลาสประเภท collection ซึ่งเป็นแบบ immutable และ read-only โดยข้อมูลภายในไม่สามารถเปลี่ยนแปลงได้หลังจากสร้าง Object แล้ว หากต้องการแก้ไขจะต้องสร้าง Object ใหม่ขึ้นมาแทน

- ไม่ใช่ Utility เพราะ Utility class มักเป็น static แต่ FrozenDictionary ต้องสร้าง instance และใช้เก็บข้อมูลจริง
- ไม่ใช่ Mutable เพราะ state ของ object คงที่ตลอดอายุการใช้งาน

### ลักษณะการใช้งาน 
ตัวอย่างคลาส:
```
FrozenDictionary<TKey,TValue>

FrozenSet<T>
```
ตัวอย่างโค้ด:
```
var frozen = myDictionary.ToFrozenDictionary();
```
- หลังจากถูกแปลงเป็น FrozenDictionary แล้วจะไม่สามารถ Add, Remove, หรือแก้ไขข้อมูลได้
- ถ้าต้องการเปลี่ยน ต้องสร้าง collection ใหม่
